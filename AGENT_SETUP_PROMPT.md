# Agent Setup Prompt

Copy/paste this into Cursor, Claude Code, Codex, or another coding agent when you want it to set up DoomGen for you.

```text
You are setting up DoomGen from a fresh clone.

Repository facts:
- DoomGen supports three modes: cloud, local, hybrid.
- Cloud mode uses OpenAI for planning/images and ElevenLabs for audio.
- Local mode uses the local FastAPI service for visuals, prefers OpenAI for planning when a real API key is configured, and otherwise falls back to Ollama.
- Hybrid mode mixes local visuals with cloud-backed planner and/or audio.
- Current MVP asset targets are:
  - weapon_sprite_set:pistol
  - enemy_sprite_set:zombieman
  - hud_patch_set:doomguy_face
  - sound_pack:pistol
- Audio requests are still provider-backed rather than generated by the local FastAPI service.

Your job:
1. Ask which mode the user wants: cloud, local, or hybrid.
2. Read `README.md`, `docs/setup-modes.md`, `.env.example`, and `package.json` before changing anything.
3. Create `.env` from `.env.example` and only fill the variables required for the chosen mode.
4. Never put real secrets into tracked files. Keep secrets in `.env` only.
5. Prefer the repo scripts over custom setup:
   - `npm install`
   - `npm run dev`
   - `npm run dev:local-mvp`
   - `npm run prepublish:audit`
   - `npm run typecheck`
   - `npm run build`
6. If the user chooses local or hybrid:
   - use Python 3.10-3.13 for `local-doomgen-service`
   - if `VITE_OPENAI_API_KEY` is set to a real key, use OpenAI for planning
   - otherwise install/start Ollama and use the configured local planner model
7. After setup, report:
   - which mode is configured
   - which env vars were required
   - what command to run next
   - any missing credentials or system dependencies

Constraints:
- Do not edit `.env.example` unless explicitly asked.
- Do not commit or print real API keys.
- Do not invent unsupported local targets.
- Use the current code behavior, not older docs or assumptions.
```
